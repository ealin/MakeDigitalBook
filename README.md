# MakeDigitalBook 📖

一個高效且穩健的 Python 書籍掃描檔 OCR 識別與數位化整合工具，專為將電子書截圖或掃描圖片自動化轉為乾淨、排版正確的文字圖書而設計。

---

## ⚡ 核心特色

1. **⚡ 超高速多模態 OCR**：
   - 採用 **Ollama 本機 HTTP API** 通訊（呼叫 `deepseek-ocr`），繞過終端機 TTY 卡死問題。
   - 單頁解析速度由數分鐘大幅縮短至 **24 秒左右**，實現極速推論。

2. **🔢 完美的檔名數字排序**：
   - 採用正規表達式動態提取檔名尾端的數值（例如 `台灣AI大未來 - 100.jpg` 提取出 `100`），避免傳統字串排序將 `10.jpg` 排在 `2.jpg` 前面的錯誤，確保整本書的文字順序 100% 正確。

3. **🧽 電子書狀態列雜訊過濾**：
   - 自動過濾電子書截圖或掃描時產生的頂部/底部狀態列雜訊，如：
     - **時間標記**（例如 `AM2:07` 或 `13:15`）
     - **剩餘電量**（例如 `73%` 或 `100%`）
     - **閱讀器 UI 選項**（例如 `AA`、單獨的箭頭等）
     - **Markdown 區塊標記**（例如 ` ```markdown `、` ``` `）與模型招呼語。

4. **🔄 斷點續傳與自動快取**：
   - 每頁處理時會自動檢查 `TXT/<book_id>/` 下是否已存在對應的 `.txt`。若已完成，則立即跳過，支援因故中斷後「無縫繼續」，省時且避免重複運算。

5. **🚨 Ollama 當機超時保護與手動重試**：
   - 當頁面處理時間過長（預設超過 180 秒）或 API 無法連接時，程式會響起終端鈴聲（Bell Sound）並進入**完全暫停狀態**。
   - 提示使用者重啟 Ollama，並在重啟完成後按下 `Enter` 鍵直接**原頁面重新發起 OCR**，確保批次處理的極高強健性。

## 🚀 使用說明

### 1. 本地自我檢測（單元測試）
在開始處理圖書之前，可以運行程式內建的測試，驗證排序演算法和清洗過濾邏輯是否完全工作正常：
```bash
python3 ./ocr_processor.py B001 --test
```

### 2. 進行圖書 OCR 處理與合併
將書籍掃描檔放置於 `SCAN_PAGES/B001_xxx` 下，接著直接執行程式：
```bash
python3 ./ocr_processor.py B001
```
*   **參數解析**：程式會自動搜尋 `SCAN_PAGES` 下以 `B001` 開頭的子目錄（如 `B001_台灣AI大未來`），依序執行 OCR 並在完成後於專案根目錄輸出合併後的書籍純文字檔 `B001.txt`。

### 3. 可用參數 (Command-Line Arguments)
```bash
python3 ./ocr_processor.py [-h] [-l LIMIT] [-t TIMEOUT] [--scan-dir SCAN_DIR] [--out-dir OUT_DIR] book_id
```
- `book_id`：必填，書籍編號（例如 `B001`）。
- `-l, --limit`：選填，指定本次運行**只處理幾頁**（如 `-l 10`），非常適合小規模測試。
- `-t, --timeout`：選填，警報超時門檻，單位為秒（預設為 180 秒）。

## 🛡️ Git 忽略規則 (.gitignore)
為保持 Git 倉儲的乾淨與輕量，以下大檔案與臨時輸出已設定為**自動忽略**：
- 所有圖片掃描檔 (`SCAN_PAGES/` 及其子目錄)
- 轉出的所有單頁純文字檔 (`TXT/` 及其子目錄)
- 轉出的整合性書籍純文字檔 (`*.txt`，如 `B001.txt`)
- 所有暫存檔案 (`temp/`)
